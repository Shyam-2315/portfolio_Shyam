# Neural Command Center Portfolio

Full-stack personal portfolio with a React/TanStack frontend, FastAPI backend, JWT-protected admin dashboard, PostgreSQL persistence, and local upload storage.

## Project Layout

```text
portfolio/
  backend/   FastAPI API, Alembic migrations, seed scripts, uploads
  frontend/  React/TanStack Start portfolio and /admin dashboard
```

## Backend Setup

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Create `backend/.env`:

```env
DATABASE_URL=postgresql+psycopg://postgres:postgres@localhost:5432/neural_command_center
JWT_SECRET_KEY=replace-with-a-long-random-secret
FRONTEND_URL=http://localhost:8080
CORS_ORIGINS=http://localhost:8080,http://127.0.0.1:8080,http://localhost:5173,http://127.0.0.1:5173
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=replace-with-a-strong-password
UPLOAD_DIR=uploads
MAX_UPLOAD_SIZE_MB=10
```

Run migrations and seed admin:

```powershell
alembic upgrade head
python seed_admin.py
```

Optional starter content:

```powershell
python seed_demo_data.py
```

Run the API:

```powershell
uvicorn app.main:app --reload
```

Swagger is available at `http://localhost:8000/docs`.

## Frontend Setup

```powershell
cd frontend
npm install
```

Create `frontend/.env`:

```env
VITE_API_BASE_URL=http://localhost:8000
```

Run the frontend:

```powershell
npm run dev
```

Open the public portfolio at the shown dev URL. Open the admin dashboard at `/admin`.

## Database Setup

Use PostgreSQL and create the database named in `DATABASE_URL`. Example:

```sql
CREATE DATABASE neural_command_center;
```

Alembic owns schema changes:

```powershell
cd backend
alembic upgrade head
```

## Admin Login

The admin dashboard uses `POST /api/auth/login`. Create the admin account with:

```powershell
cd backend
python seed_admin.py
```

The script reads `ADMIN_EMAIL` and `ADMIN_PASSWORD` from `backend/.env`. The frontend stores the returned JWT in local storage and attaches it as a Bearer token through `src/lib/api.ts`.

## Uploads

Uploads are stored under `backend/uploads/` by category:

- `uploads/image/`
- `uploads/note/`
- `uploads/resume/`

FastAPI serves this folder at `/uploads`, so a saved value such as `/uploads/image/file.png` renders in the frontend as `http://localhost:8000/uploads/image/file.png`.

Allowed upload types:

- Images: JPEG, PNG, WebP, GIF
- Notes: PDF, TXT, Markdown, DOC, DOCX
- Resume: PDF

`MAX_UPLOAD_SIZE_MB` controls the file size limit.

## API Smoke Test

With the backend running:

```powershell
cd backend
$env:API_BASE_URL="http://localhost:8000"
$env:ADMIN_EMAIL="admin@example.com"
$env:ADMIN_PASSWORD="replace-with-a-strong-password"
python api_smoke_test.py
```

The smoke test checks public GET access, protected write access without a token, invalid ID behavior, login, and upload validation.

## Common Errors

- `401 Not authenticated`: Login again from `/admin`; the JWT is missing or expired.
- `Invalid email or password`: Run `python seed_admin.py` after setting `ADMIN_EMAIL` and `ADMIN_PASSWORD`.
- CORS blocked request: Add the frontend origin to `CORS_ORIGINS` in `backend/.env`.
- Upload returns `400 Unsupported file type`: Use one of the allowed MIME types listed above.
- Upload returns `413 File too large`: Increase `MAX_UPLOAD_SIZE_MB` or upload a smaller file.
- Public portfolio shows empty states: Seed demo data or add real records from `/admin`.

## Deployment Notes

Use the example files as templates:

- Backend: `backend/.env.production.example`
- Frontend: `frontend/.env.production.example`

Never commit real `.env` files. Keep production values in the deployment provider's environment variable UI.

### Frontend On Vercel

Recommended settings:

- Root directory: `frontend`
- Install command: `npm install`
- Build command: `npm run build`
- Output: use the framework defaults generated by TanStack/Vite
- Environment variable: `VITE_API_BASE_URL=https://your-backend-domain`

After deployment, copy the Vercel frontend URL into backend production env:

```env
FRONTEND_URL=https://your-frontend.vercel.app
CORS_ORIGINS=https://your-frontend.vercel.app
```

### Backend On Render Or Railway

Recommended backend settings:

- Root directory: `backend`
- Install command: `pip install -r requirements.txt`
- Start command:

```sh
alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port $PORT
```

Required environment variables:

```env
ENVIRONMENT=production
DATABASE_URL=postgresql+psycopg://USER:PASSWORD@HOST:5432/DATABASE?sslmode=require
JWT_SECRET_KEY=replace-with-a-long-random-production-secret
FRONTEND_URL=https://your-frontend.vercel.app
CORS_ORIGINS=https://your-frontend.vercel.app
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=replace-with-a-strong-production-password
UPLOAD_DIR=uploads
MAX_UPLOAD_SIZE_MB=10
```

Health check endpoint:

```text
GET /health
```

Use `https://your-backend-domain/health` as the Render/Railway health check URL.

### PostgreSQL On Neon Or Supabase

Create a managed PostgreSQL database, then set `DATABASE_URL` in the backend service. Use the provider's pooled or direct connection URL as appropriate for your deployment. Keep SSL enabled, usually with `sslmode=require`.

Run migrations during backend startup with:

```sh
alembic upgrade head
```

Create the first admin user after environment variables are set:

```sh
python seed_admin.py
```

### CORS Setup

Development defaults allow common localhost origins:

```env
CORS_ORIGINS=http://localhost:3000,http://localhost:5173,http://localhost:8080,http://127.0.0.1:5173,http://127.0.0.1:8080
```

Production should allow only the deployed frontend origin:

```env
FRONTEND_URL=https://your-frontend.vercel.app
CORS_ORIGINS=https://your-frontend.vercel.app
```

### Upload Persistence Warning

Local uploads are stored under `backend/uploads/`. On Render/Railway, local filesystem uploads are not permanent unless you configure persistent disks/volumes. Deployments, restarts, or container rebuilds can remove uploaded files.

For production portfolio media, migrate uploads to durable object storage.

- `.env` and `.env.*` are ignored, except example files.
- Do not commit real `DATABASE_URL`, `JWT_SECRET_KEY`, `ADMIN_PASSWORD`, or provider API keys.
- Rotate credentials immediately if a real secret is accidentally committed.
